# Feature 06: Cross-Model Review

**Repository:** imas-codex  
**Complexity:** Medium  
**Depends on:** 05 (SN Build Pipeline)  
**Enables:** 08 (Publish)  
**Wave:** 4

---

## Overview

Implement a "rubber duck" review phase where candidates generated by Model A are reviewed by Model B (different LLM family). This catches systematic biases and hallucinations that self-review cannot detect.

## Design

### Review Strategy
- Generate with one model family (e.g., Claude), review with another (e.g., GPT)
- Each candidate reviewed for:
  - Grammar correctness (does the name parse?)
  - Semantic accuracy (does the name match the source quantity?)
  - Description quality (clear, concise, accurate?)
  - Unit consistency (does the unit match the source data?)
  - Dedup check (is there an existing catalog entry that covers this?)

### Review Output
- Accept / Reject / Revise decisions
- Confidence score (0-1)
- Revision suggestions for marginal candidates
- Structured via `acall_llm_structured()` with Pydantic review model

### Integration with Pipeline
- Plugs into the REVIEW phase of Feature 05
- Uses the same `WorkerSpec` / `PipelinePhase` pattern
- Configurable: `--review-model MODEL` CLI option
- Can be disabled for benchmarking runs: `--skip-review`
